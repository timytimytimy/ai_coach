# 智能力量训练教练 App（UI 设计稿 v1）

## 1. 视觉定位
- 关键词：可信、克制、运动科学、教练记录
- 主题：深色仪表盘（训练房氛围）+ 单一强调色（用于 CTA/严重度/趋势）

## 2. 设计 Tokens（v1）
### 2.1 颜色（Dark）
- `bg/0` #0B0F14
- `bg/1` #101723
- `surface/0` #131C2A
- `surface/1` #172235
- `border/0` #223149
- `text/primary` #EAF0FA
- `text/secondary` #A9B4C7
- `text/tertiary` #7E8AA1
- `accent/primary` #3B82F6
- `danger` #EF4444
- `warn` #F59E0B
- `success` #22C55E

严重度映射（同一套视觉）：
- `severity/high` = `danger`
- `severity/med` = `warn`
- `severity/low` = `accent/primary`

### 2.2 字体与字号
- 字体：
  - 标题：`Space Grotesk`（600）
  - 正文/UI：`IBM Plex Sans`（400/500）
  - 数字/指标：`IBM Plex Mono`（400/500，tabular风格）
- `H1` 24/30 Semibold
- `H2` 18/24 Semibold
- `Body` 15/22 Regular
- `Caption` 12/16 Regular
- 数字：使用等宽/表格数字（指标列对齐）

### 2.3 间距与圆角
- 间距：`4, 8, 12, 16, 20, 24`
- 圆角：卡片 `12`，小胶囊 `999`
- 分割线：`1px`，颜色 `border/0`

## 3. 信息架构（4 Tab）
- `训练`：今天怎么练（入口）
- `分析`：每个 set 的结果与对比
- `计划`：下一节课处方 + 调整解释
- `我的`：目标/标定/拍摄/隐私

## 4. 核心组件（规格）
### 4.1 `ConfidenceBadge`
- 状态：`高可信` / `可用但不完整` / `不可用`
- 颜色：
  - 高可信：`success` 底色 12% 透明
  - 不完整：`warn` 底色 12%
  - 不可用：`danger` 底色 12%

### 4.2 `FindingCard`（固定结构）
- 顶行：`问题名` + `SeverityPill`
- 证据行：`Rep # / 时间区间 / 指标值`
- 影响行：1 句话（最多 24 字）
- 纠正行：1–2 条 cue（短句）

### 4.3 `MetricPill`
- 文案：`Avg V 0.42 m/s` / `VL 18%` / `Score 78`
- 点击：展开说明（定义/单位/何时关注）

### 4.4 `VelocityChart`
- 默认：仅高亮向心段；离心段灰
- 交互：点选 rep → 显示该 rep 的 Avg/Peak

### 4.5 `Primary CTA`
- 形态：全宽按钮，高 48，圆角 12，底色 `accent/primary`
- 文案：动词开头（开始训练/开始录制/保存并分析）

## 5. 关键页面（布局骨架）
### 5.1 `训练`首页
- 顶部：今日日期 + 状态（可选：睡眠/酸痛）
- 卡1：`下一节课处方`（3–5 个 set 列表，显示动作/重量/reps）
- 主 CTA：`开始训练`
- 卡2：`最近一次建议`（1 张 FindingCard 预览 + “去复训”）

### 5.2 `记录 Set`
- 表单：动作（SQ/BP/DL）→ 重量 → reps → RPE（可选）
- 视频来源：`开始录制`（主 CTA）/ `从相册导入`（次 CTA，用于补录与兜底）
- 录制/导入前底部弹层：`侧面 / 全身入镜 / 相机固定 / 建议 30–45s`
- 录制结束或导入完成：进入 `上传与分析中`（可返回训练继续记录）

### 5.3 `上传与分析中`（异步核心）
- 展示：阶段进度（上传/排队/分析）+ 原因标签（如网络失败/队列拥堵）
- 主 CTA：`继续记录下一组`
- 次 CTA：`重新上传` / `更换视频`

### 5.4 `Set 结果页`（主战场）
- 顶部：`Score 78` + `ConfidenceBadge` + `3 个关键指标 Pill`
- 区块 A：Top 3 `FindingCard`
- 区块 B：`全部问题清单`（默认折叠，条目显示 `mm:ss–mm:ss + 问题名 + 严重度`）
- 区块 C：`处方`（1 个 drill + 训练量建议）
- 区块 D：`速度`（可用时显示 VelocityChart + VL + Stop Set 建议；不可用时显示原因）
- 区块 E：`证据回放`（叠加回放入口，支持跳转到 `mm:ss–mm:ss`）
- 底部固定：反馈条（有用/没用/不确定/疼痛）

### 5.4 `叠加回放`
- 播放器 + 开关：关键点 / 杠铃轨迹 / rep 边界
- 红色高亮区间：与 findings evidence 对齐

### 5.5 `计划`
- 卡1：`下一节课怎么练`（可执行）
- 卡2：`本次自动调整原因`（基于 VL/RPE 的一句话解释 + 数值）
- 折叠：计划历史与回滚

## 6. 文案库（直接可用）
- 不可用：`本次视频无法可靠分析：{原因}。建议按“侧面/全身入镜/相机固定”重新录制。`
- 不完整：`动作分析可用，但速度未能计算：{原因}。你仍可参考技术问题与处方。`
- 高可信提示：`本次分析可信度高，可用于对比趋势与训练调整。`
- Stop Set：`当速度损失达到 {VL}% 建议停止本组，优先保证动作质量。`
- 疼痛反馈：`如果出现疼痛，请停止相关动作训练并寻求专业人士建议。`
