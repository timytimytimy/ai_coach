# 🏋️ VBT驱动关键帧采样系统 PRD

## 一、项目背景

本系统用于从力量举视频（深蹲 / 硬拉）中自动提取高价值关键帧，用于后续人工标注和模型训练。

当前已有能力：

* 视频数据
* 杠铃轨迹 / VBT（速度）检测能力

目标是：

> 用最少的帧，覆盖最关键的动作信息，提升标注效率和数据质量

---

## 二、系统目标

输入：

* 视频文件（或视频帧序列）
* 杠铃轨迹数据（时间序列）
* VBT数据（速度、加速度）

输出：

* 关键帧图片（jpg/png）
* 每帧附带元信息（JSON）

---

## 三、核心设计原则

1. **信息密度优先**

   * 不做均匀抽帧
   * 只选关键动作阶段

2. **VBT驱动**

   * 利用速度/加速度信号定位关键帧

3. **避免重复**

   * 同一rep最多抽5帧

4. **多样性覆盖**

   * 控制不同视频的采样比例

---

## 四、系统流程

```text id="flow1"
视频输入
  ↓
VBT数据读取
  ↓
动作分段（rep segmentation）
  ↓
每个rep关键帧提取
  ↓
去重与筛选
  ↓
保存图片 + metadata
```

---

## 五、输入定义

### 1. 视频

支持：

* mp4 / mov

---

### 2. VBT数据（必须）

每帧包含：

```json
{
  "frame_index": int,
  "time": float,
  "bar_y": float,
  "velocity": float,
  "acceleration": float
}
```

说明：

* bar_y：杠铃垂直位置
* velocity：垂直速度
* acceleration：速度变化率

---

## 六、Rep分段

目标：

* 将视频拆分成多个动作（rep）

### 方法（建议）

基于：

* bar_y 的周期变化
* velocity 的符号变化（上升 / 下降）

### 输出：

```json
{
  "rep_id": 1,
  "start_frame": 100,
  "end_frame": 240
}
```

---

## 七、关键帧采样策略（核心）

## 每个 rep 抽 5 帧：

---

### 1. Bottom（最低点）

定义：

```text
bar_y 最小值
```

---

### 2. v_min（最低速度点）

定义：

```text
velocity 最小值（上升阶段）
```

说明：

* 通常是 sticking point

---

### 3. dv/dt 最大（变化最剧烈）

定义：

```text
|acceleration| 最大
```

用途：

* 检测动作变化
* 出底瞬间

---

### 4. Setup（起始帧）

定义：

```text
rep开始前5~10帧
```

要求：

* 杠铃静止
* 动作尚未开始

---

### 5. 随机帧（1个）

目的：

* 提高泛化
* 防止模型只学关键帧

---

## 八、去重机制

避免抽到“几乎相同帧”

规则：

```text
相邻关键帧间隔 ≥ N 帧（默认10）
```

如果冲突：

* 保留优先级高的帧：

优先级：

```text
v_min > bottom > dv/dt > setup > random
```

---

## 九、多样性控制

防止某个视频占比过高：

### 策略：

* 每个视频最多抽：

  * max_frames_per_video（默认30）

* 如果超过：

  * 按rep均匀采样

---

## 十、输出设计

## 1. 图片输出

```text
output_frames/
  video1_rep01_bottom.jpg
  video1_rep01_vmin.jpg
  ...
```

---

## 2. metadata JSON

```json
{
  "video": "video1.mp4",
  "rep_id": 1,
  "frame_index": 132,
  "type": "v_min",
  "time": 3.42,
  "velocity": -0.21,
  "acceleration": 0.85
}
```

---

## 十一、配置参数

支持配置：

```json
{
  "frames_per_rep": 5,
  "min_frame_gap": 10,
  "max_frames_per_video": 30,
  "random_seed": 42
}
```

## 十二、异常处理

需要处理：

* VBT缺失
* rep分段失败
* 视频损坏
* 帧读取失败

行为：

* 跳过异常rep
* 记录日志
