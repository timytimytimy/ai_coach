# 智能力量训练教练 App（Design Review v1）

## 0. 设计目标与原则
这是一个“信任型”产品。设计的首要目标：让用户相信结论、知道怎么改、并能在下一次训练看到改善。

产品约束（最新）：离线上传视频 → 云端异步生成报告（CPU-only）。报告要求能定位到 `mm:ss-mm:ss` 的证据区间，并且“列出全部问题，但只详细讲解 Top3”。

MVP 设计原则：
- 结果页信息层级固定：`Top3 详解` + `全部问题清单（折叠）` + `1 个处方` + `1 个下次调整（含解释）`
- 每条结论都必须带“证据”（指标数值 + rep（如有）+ `mm:ss-mm:ss`），并可回放核验
- 不确定性必须可视化（可用性/置信度/失败原因），不可硬输出
- 异步是默认：允许用户“先去训练/记录下一组”，报告完成后再回看

## 1. 品牌与视觉方向
- 气质：专业、克制、偏运动科学与教练笔记风格
- 颜色：深色/中性色为主 + 1 个高对比强调色（严重度、CTA、上升趋势）
- 排版：数字对齐、网格化布局、留白充足，强调“仪表盘可信度”

## 2. 信息架构（MVP）
建议 4 Tab：
- `训练`：今日训练入口、开始记录、最近建议
- `分析`：按 set 浏览报告（筛选 SQ/BP/DL）
- `计划`：本周计划与下一节课处方、调整历史
- `我的`：目标、标定、拍摄规范、隐私与数据

对象层级：`Day → Workout → Set → Report → (FindingEvent / Rep / Overlay)`
- 绝大多数用户停留在 `Set/Report` 层
- `Overlay` 仅用于核验证据（关键点/杠铃轨迹/rep边界）

异步状态是核心对象：`Set` 绑定 `ReportJob`（queued/running/succeeded/failed/partial）。任何设计必须先写清楚“用户在每个状态看到什么”。

## 3. 关键页面规格（逐页）

### 3.1 `训练首页`
目标：快速开练，减少输入。
- 今日计划卡：下一节课的 3–5 个 set 概览
- 主按钮：`开始训练/开始录制`
- 次卡：最近一次分析结果（1 张卡）提示复训要点

### 3.2 `记录 Set`（表单）
目标：最少输入完成一组记录，并把拍摄规范变成默认行为。
- 字段顺序：动作 → 重量 → 目标 reps → 实际 reps → RPE（可选）→ 绑定视频
- 视频来源：
  - 默认：`录制视频`（主入口）
  - 可选：`从相册导入`（补录/历史视频/相机权限问题兜底）
- 录制/导入前强提示：`侧面/全身入镜/相机固定` + `建议 30–45 秒`
- 完成后：进入 `上传与分析中` 状态页（允许返回训练继续记录下一组）

### 3.3 `上传与分析中`（异步核心页）
目标：让用户在等待时仍能继续训练，并始终知道“现在进行到哪一步/是否可用”。
- 状态：`上传中` → `排队中` → `分析中` → `已完成/部分完成/失败`
- 进度展示：显示当前阶段（转码/姿态/轨迹/切rep/生成报告）与预计剩余（粗略即可）
- 主动作：`继续记录下一组`（优先）
- 次动作：`留在此页等待` / `后台完成后通知我`
- 操作（必须有）：
  - `重新上传`：网络失败/上传中断时显式重试
  - `更换视频`：导入错误视频或录错时快速替换
- 失败态：展示原因 + `如何重新拍摄/重新上传`，并允许仅查看已有的部分结果（若 partial）

### 3.4 `Set 报告页（最重要）`
目标：用“证据 + 可执行建议”建立信任，同时满足“全量问题清单 + Top3详解”。

顶部区：
- 综合评分（0–100）
- `ConfidenceBadge`：高可信 / 可用但不完整 / 不可用
- 本次上下文：重量、reps、RPE（若有）

主区（固定结构，避免信息爆炸）：
- `Top3 详解`：3 张 `FindingCard`（每卡固定结构）
  - 标题：问题名 + 严重度
  - 证据：指标数值 + `mm:ss-mm:ss`（如能对齐 rep，则显示 Rep #）
  - 影响：一句话说明为什么重要
  - 纠正：1–2 条短口令 cue
- `全部问题清单`（默认折叠）：按严重度分组，行内只显示 `mm:ss-mm:ss + 问题名 + 严重度`，点开某条可跳转对应回放区间

处方区：
- `PrescriptionCard`：只给一个最推荐 drill（例如暂停/节奏/辅助动作）

速度区（可用时展示，不可用则显示原因而非直接消失）：
- 速度曲线（默认只画向心段，离心段灰色）
- 指标：平均/峰值/速度损失
- `StopSetSuggestion`：停组建议（例如达到 20% 损失则停）

证据回放：
- `叠加回放`入口：关键点/杠铃轨迹/rep 边界（跳转到指定 `mm:ss-mm:ss`）

反馈闭环（固定在底部）：
- `这条建议有用吗？`：有用/没用/不确定/引发疼痛

### 3.5 `叠加回放`（面向信任的证据页）
目标：把“为什么这么判”展示给用户，降低争议。
- 开关：关键点、杠铃轨迹、rep 边界、慢放
- 高亮区间：与报告中的 `mm:ss-mm:ss` 对应；从任意问题条目一键跳转
- 占位与降级：若某模块缺失（例如杠铃轨迹不可用），对应开关置灰并解释原因

### 3.5 `计划页`
目标：只呈现“下一节课怎么练”，并解释自动调整原因。
- 本周计划概览：周视图（训练日卡片）
- 下一节课处方：动作/重量/组次/reps/RPE目标
- 调整解释：`本次速度损失/ RPE → 下次重量变化`
- 历史折叠：版本回滚入口（Pro 或高级功能）

### 3.6 `标定与拍摄规范（我的页）`
目标：提升分析可用率，降低“算不出来”的挫败。
- 标定：杠铃片直径/参考物（一次配置，多次复用）
- 拍摄规范：一张示意图 + 三条规则（侧面、全身、相机固定）
- 失败示例：遮挡/角度错误/相机晃动，说明会导致什么结果

## 4. 组件库（设计系统 v1）
- `FindingCard`：严重度色条 + 标题 + 证据小字 + cue
- `MetricPill`：短指标（可点开解释与单位）
- `ConfidenceBadge`：高可信/可用但不完整/不可用
- `Chart`：速度曲线、趋势曲线（7/30天）
- `CTA Button`：主CTA/次CTA/危险操作（删除数据）

严重度视觉映射：
- `high`：强强调色 + 高对比
- `medium`：中性强调
- `low`：弱化显示，默认折叠

## 5. 文案规范（信任与安全）
- 结论措辞：使用“训练建议”而非“诊断”
- 低置信度：必须说明原因（角度/遮挡/追踪失败）+ 给出补录建议
- 指标解释：MetricPill 展开后说明“怎么计算/代表什么/何时关注”
- 疼痛反馈：引导用户停止相关动作并建议咨询专业人士（非医疗结论）

## 6. 交互状态覆盖（必须写进计划）
| 功能/页面 | LOADING | EMPTY | ERROR | SUCCESS | PARTIAL |
|---|---|---|---|---|---|
| 上传与分析中 | 阶段进度+可返回训练 | 无视频/未绑定set提示+去录制 | 上传失败/排队过长/超时（可重试） | 跳转报告页 | 显示已生成部分并解释缺失原因 |
| Set 报告页 | 骨架屏 + 占位文案 | 无问题时：说明“本组动作稳定”，给下一步 | 报告生成失败：原因+重试+拍摄建议 | Top3详解+全量清单 | 缺速度/缺轨迹/缺pose：模块置灰+原因 |
| 叠加回放 | 加载进度 | 无可用叠加：只给原视频回放 | 回放资源加载失败 | 可跳转到证据区间 | 仅部分叠加可用 |
| 复训对比 | 加载对比对象 | 没有可对比历史：引导先完成一次 | 拉取失败/权限问题 | 上次vs本次对比卡 | 仅评分可比、速度不可比 |
| 计划调整解释 | 加载 | 无计划：引导创建模板 | 生成失败：提示手动输入 | 展示因果链与数值 | 缺VBT时用Pose/训练负荷解释 |

## 7. 不确定性与失败降级（必须实现）
- `不可用`：直接给拍摄指导，不展示评分与结论
- `可用但不完整`：展示技术评分与部分 findings，并明确哪些模块缺失（例如速度）
- `高可信`：展示完整 findings + 速度 + 疲劳建议

## 8. 响应式与无障碍（MVP底线）
- 触控目标：最小 44px；底部反馈条与主要 CTA 不能过小
- 对比度：深色主题下 `text/secondary` 与背景需满足可读性（至少达到 WCAG AA 的视觉效果标准）
- 动作/指标的单位与数字对齐：支持大字号（系统字体放大时不溢出关键数值）
- 语义化：严重度与状态不只靠颜色，必须有文本标签（高/中/低、可用/不可用）

## 9. 未决设计决策（实现前必须定）
| 决策 | 如果不定会发生什么 |
|---|---|
| 报告页的“全量问题清单”默认展开还是折叠 | 工程师随意选，信息密度失控 |
| 报告完成的通知方式（仅站内/推送/短信） | 用户不知道结果已好，留存变差 |
| 失败原因文案规范（枚举→用户可理解） | 出现“unknown error”，信任崩 |
