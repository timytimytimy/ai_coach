# 智能力量训练教练 App（CEO Pitch v1）

## 1. 一句话
用手机视频把深蹲/卧推/硬拉的每一组训练量化：动作问题（带证据）+ 杠铃速度（VBT）+ 疲劳管理 → 自动给出下一次训练怎么练。

## 2. 目标用户
- 力量训练进阶者/力量举爱好者：有计划训练习惯、愿意为提升与更少受伤风险付费。
- 有教练但缺少“课外高频反馈”的训练者：想要每组都被监督与对比。

## 3. 痛点
- 自我训练很难发现关键技术问题，纠错成本高。
- 速度与疲劳只能凭感觉，容易“练过头”或“练不够”。
- 私教/线上教练反馈频率有限，VBT硬件门槛高。

## 4. 解决方案（闭环）
1) 选择动作/重量/目标（力量或技术）
2) 录制一组（Set）
3) 端侧分析输出：
- Top 3 动作问题（问题→影响→纠正提示→优先级）
- 证据：哪一rep/哪一段、对应指标数值
- 杠铃速度：速度曲线、平均/峰值速度、速度损失（Velocity Loss）
4) 自动训练总结：当日负荷与疲劳状态
5) 下次训练建议：重量/组数/停止点与微周期调整

## 5. 核心差异化（护城河）
- 数据资产不是“视频”，而是 `rep级`：关键点时序 + 杠铃轨迹 + 速度指标 + 训练上下文 + 用户后验结果（下次是否改善）。
- 可回归验证的指标/规则体系：每次迭代都能在金标集上评测，不靠黑箱提示词。
- 端侧推理为主：低成本、隐私友好、反馈更即时。

## 6. MVP范围（8–10周）
必做：
- 训练日志：动作、重量、组次、reps、RPE/RIR
- 视频录制与回放（按组绑定）
- 三大项基础技术评分（每项 3–5 个可解释指标）
- 速度模块：单视角侧面 + 半自动标定 → 速度曲线与速度损失
- 疲劳：速度损失 + 近7天训练负荷（sRPE/tonnage）
- 下次训练调整：规则化加减重量/组数/停组建议
暂缓：多机位/全自动标定、复杂多周计划生成、社交与穿戴设备深集成。

## 7. 数据与AI策略（可规模化且合规）
- 合规来源：授权素材 + CC许可公开视频；尽量只存派生特征（关键点/轨迹/指标），保留来源与许可。
- 三层体系：
  - 指标/规则层：结构化“问题→观测量→阈值/模式→严重度→证据→处方”，可版本化。
  - 金标集：每动作先 200–500 个 rep 人工复核，用于回归评测。
  - LLM层（Gemini）：只负责解释与个性化表达，必须引用证据字段，不承担主判断。

## 8. 技术架构（推荐默认）
- App：Flutter
- 端侧推理：MediaPipe Pose（人体关键点为主，后续可迁移到更稳的 top-down pose 方案）
- 服务端：用户/日志/计划版本、分析结果存储、配置下发（PostgreSQL + 对象存储）
- 后期AI服务：批处理/更重的检测（队列 + GPU推理）

## 9. 商业模式
- 订阅：Basic（分析与趋势）/ Pro（速度+疲劳+计划自动调整）
- 增值：教练复核包（按次或月包），提升信任与ARPU。

## 10. 关键KPI（必须盯）
- 分析可用率：满足拍摄门槛后能出结果 ≥ 85%
- 信任：建议采纳率 ≥ 40%，否定率持续下降
- 留存：W1 ≥ 25%（核心人群内测）
- 成本：端侧为主，控制单用户月推理成本

## 11. 12个月路线图
- 0–2个月：MVP闭环（录制→分析→建议→下次调整→趋势复盘）
- 3–6个月：质量门控与降级策略、金标集与评测流水线、计划模块自适应
- 7–12个月：动作族扩展、教练生态、标定自动化与多机位（提升速度准确性与可用率）

## 12. 30秒对外话术
我们做的是力量训练的智能教练。用户用手机录制深蹲/卧推/硬拉的一组动作，App会给出最关键的动作问题、杠铃速度曲线和疲劳状态，并自动调整下一次训练处方。我们把每一次rep变成可量化的数据资产，用可解释、可评测的指标体系保证稳定性，同时让AI用教练语言把建议讲清楚，让进阶训练者以更低成本获得接近私教级的高频反馈与计划管理。
